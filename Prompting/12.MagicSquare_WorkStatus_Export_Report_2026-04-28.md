## 2. Dual-Track TDD와의 대응

- **Track A / B를 한 PR에** 넣을지, **A 먼저 B 나중**으로 쪼갤지는 팀/선호에 따라 달리면 됩니다.  
- 원칙: **한 PR에 “한 가지 의미 있는 단위”** — 예: FR-01(경계) + 그에 맞는 TC-04~07 RED→GREEN이 한 세트.
- `feature/…`가 커지면 **PRD의 US/FR 단위로** 더 쪼개는 편이 리뷰·회귀에 유리합니다.

## 1. 기본 축: `main` + 짧은 작업 브랜치

## 2. 단계별 브랜치 이름 (추천)

PRD·README의 **FR / US / Track**과 맞추면 추적이 쉽습니다.

**① Dual-Track UI + Logic TDD (RED)**  
- 의미: Track A·B에서 **실패하는 테스트**를 같이/교차로 추가하는 단계.  
- 브랜치 예:  
  - `feature/fr-01-boundary-red`  
  - `feature/fr-02-blanks-red`  
  - 또는 한 스프린트 단위: `feature/us-001-io-domain-red`  

같은 주제에서 A/B를 **한 브랜치**에 두면 PRD **§10.3** (I/O RED & Logic RED 병행)과 잘 맞습니다.

**② GREEN**  
- 선택 1: RED 브랜치에서 **그대로 구현**해 GREEN까지 올리고 `main`에 머지 (가장 단순).  
- 선택 2: 규모가 크면 `feature/...-red`에서 `feature/...-green`으로 **이름만 바꾸거나** PR 제목에 `[GREEN]` 표기 (브랜치를 꼭 나누지 않아도 됨).

**③ Dual-Track Refactor**  
- **동작(테스트)을 바꾸지 않는** 정리는 같은 브랜치에서 커밋하거나,  
- `refactor/fr-01-boundary` 처럼 **REFACTOR 전용** 짧은 브랜치를 `main`에서 분기해 **리팩터만** 넣고 머지.  
- 리팩터 후 **전체 `pytest` + (정책이면) 커버리지**로 GREEN 유지를 확인.

## 3. 권장 워크플로우 (한 사이클)

1. `main`에서 `feature/<scope>-red` 생성.  
2. **Dual-Track**: Boundary(Track A) 테스트 + Entity(Track B) 테스트를 PRD·TC에 맞게 **RED**로 추가.  
3. **GREEN**: 최소 구현 → 전부 통과.  
4. **Refactor**: 경계/도메인/오케스트레이터 정리, **테스트 그대로**.  
5. PR → `main` 머지 (squash 또는 merge는 팀에 하나로 통일).  
6. 다음 FR/US는 **새 `feature/...` 브랜치**로 반복.

`red`라는 **고정 브랜치 하나**에만 계속 쌓기보다, **FR·US마다 `feature/...`를 새로** 쓰는 편이 히스토리·리뷰에 유리합니다. (이전에 쓰신 `red`는 “첫 RED”용으로만 써도 됨.)

---

## 4. 병행·병합 시 주의 (Dual-Track)

- **Track A / B**를 **서로 다른 장기 브랜치**에 오래 갈라두면 머지 비용이 큼 → **같은 주기·같은 PR**에 가깝게 유지.  
- **Refactor**는 “트랙 나누기”가 아니라 **한 번에 전체 레이어**를 깨끗이 (ECB, NFR-05) 맞추는 커밋으로 묶으면 diff 이해가 쉬움.

**Git branch(브랜치)**는 커밋 줄기를 나누는 **이름 붙은 포인터**입니다. 같은 저장소 안에서 `main`에서 기능만 따로 개발한다든지, 실험용으로 잠깐 갈라놓을 때 씁니다. 브랜치를 바꾼다 = **어느 커밋을 작업의 기준으로 볼지** 바꾼다는 뜻에 가깝습니다.

- **`main`**: 보통 기본·배포·안정 버전 줄기로 씀.  
- **다른 브랜치** (예: `red`, `feature/...`): 그 위에서만 커밋하다가 나중에 `main`에 **merge**하거나 **PR**로 합침.

로컬에 어떤 브랜치가 있는지 보려면 터미널에서:

`git branch` — 로컬 목록 (현재 브랜치에 `*`)  
`git branch -a` — 원격까지 포함

이 Magic Square 저장소에서는 이전에 **`main`**과 **`red`** 두 가지를 쓰셨습니다.
