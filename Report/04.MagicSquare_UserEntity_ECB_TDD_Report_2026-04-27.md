# MagicSquare User 엔티티 구현 보고서

## 작업 개요
- 목적: `.cursorrules` 규칙을 준수하여 ECB 패턴의 `User` 엔티티와 테스트를 작성
- 기준 규칙:
  - 아키텍처: ECB (`boundary -> control -> entity`)
  - 코드 스타일: PEP8, 타입힌트 필수, Google docstring
  - 테스트: `pytest`, AAA 패턴
  - 금지사항: `print()` 디버깅 금지

## 생성/수정 파일
- `src/entity/user.py`
- `tests/entity/test_user.py`
- `src/__init__.py`
- `src/entity/__init__.py`
- `tests/__init__.py`
- `tests/entity/__init__.py`

## 엔티티 설계 요약
- `User`는 `entity` 레이어의 순수 도메인 객체로 구현
- 불변(immutable) 중심 설계:
  - `@dataclass(frozen=True, slots=True)` 적용
  - 상태 변경은 `rename`, `activate`, `deactivate`에서 새 인스턴스 반환
- 도메인 검증 규칙:
  - `user_id` 공백 금지
  - `display_name` 공백 금지
  - `preferred_input_mode`는 `ui | api | cli`만 허용
- 외부 의존성/UI 로직 없음 (ECB 경계 준수)

## 테스트 설계 요약 (AAA)
- 대상: `tests/entity/test_user.py`
- 시나리오:
  - 정상 생성 값 검증
  - 빈 `user_id` 예외 검증
  - 빈 `display_name` 예외 검증
  - 잘못된 `preferred_input_mode` 예외 검증
  - `rename` 불변성 검증 (원본 미변경, 새 객체 변경)
  - `deactivate/activate` 상태 전환 검증
