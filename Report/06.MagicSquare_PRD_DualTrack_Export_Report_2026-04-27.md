## 4) PRD 핵심 구성 요약

- 고정 계약:
  - 입력: 4x4 `int[][]`, 0은 빈칸, 빈칸 2개, 값 범위(0 또는 1~16), 0 제외 중복 금지, row-major 첫 빈칸 정의
  - 출력: `int[6] = [r1,c1,n1,r2,c2,n2]`, 1-index 좌표, 두 조합 시도 규칙 고정
- 기능 요구사항:
  - FR-01 입력 검증(Boundary)
  - FR-02 빈칸 탐색
  - FR-03 누락 숫자 탐색
  - FR-04 마방진 판정
  - FR-05 해 찾기(두 조합 시도)
- 도메인 규칙(BR-01~BR-12):
  - 크기/범위/중복/row-major/합(34)/출력 포맷/결정론/입력 불변성 규칙 명시
- 비기능 요구:
  - Domain 95%+, Boundary 85%+ 커버리지
  - 결정론 보장
  - 입력 원본 불변
  - 4x4 기준 50ms 이내 처리 목표
- Dual-Track TDD:
  - Track A(Boundary) 계약 우선 테스트
  - Track B(Domain) 불변조건/메서드 단위 테스트
  - 병렬 규칙: UI RED & Logic RED -> UI GREEN & Logic GREEN -> REFACTOR

## 5) 오류/실패 정책 확정 사항

- 입력 오류: `E_UI_MATRIX_SIZE`, `E_UI_BLANK_COUNT`, `E_UI_VALUE_RANGE`, `E_UI_DUPLICATE`, `E_UI_OUTPUT_FORMAT`
- 해 없음(두 조합 실패): Domain `SolveFailure` 발생, Boundary `E_DOMAIN_NO_SOLUTION` 매핑
- 메시지 정책: 고정 문자열 완전 일치 비교 테스트 적용
