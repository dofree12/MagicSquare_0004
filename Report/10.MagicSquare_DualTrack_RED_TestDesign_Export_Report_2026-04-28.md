## 1. 목적

- 시나리오를 **실패하는 테스트(RED)**로 변환하기 위한 **Track A / Track B** 설계를 Report에 고정한다.  
- **구현 코드·GREEN·REFACTOR는 범위 밖**이며, 본 문서는 **RED에 필요한 Given/When/Then·계약·Invariant**만 기술한다.  
- 도메인 함수명은 설계 편의상 `find_blank_coords`, `find_not_exist_nums`, `is_magic_square`, `solution`을 사용한다. (PRD §12·§14 컴포넌트명과 추후 1:1 매핑.)

---

## 2. 프로젝트 조건 (요약)

| 구분 | 내용 |
|------|------|
| 입력 | 4×4 `int[][]`, `0`은 빈칸, **정확히 2개**, 값 `1~16`, **0 제외 중복 없음** |
| 출력 | `int[6]`, **1-index** `[r1, c1, n1, r2, c2, n2]` |
| 마방진 상수 (4×4) | **34** |

---

## 3. Dual-Track RED 구조

```
UI RED (Track A)     Logic RED (Track B)
      ↓                      ↓
   (둘 다 실패 상태, 구현 전)
```

---

## 4. TRACK A — UI / Boundary RED

전제: 외부 입력 `int[][]`와 **경계 응답**(검증 실패 시 예외 또는 `{ code, message }` 등 고정 계약). Domain 구현에 의존하지 않음.

### UI-A-01 — 행렬 크기가 4×4가 아님

| 항목 | 내용 |
|------|------|
| **테스트 이름 (예)** | `test_rejects_non_4x4_matrix_size` |
| **Given** | `3×4` 또는 `4×3` 등 행·열이 4가 아닌 정수 배열. |
| **When** | Boundary 입력 검증(또는 이를 감싼 최소 진입점) 호출. |
| **Then** | **`E_UI_MATRIX_SIZE`**에 대응하는 실패(예외 또는 동일 의미 오류 객체). |
| **보호 계약** | **BR-01 / AC-FR01-01**: 외부 입력은 반드시 4×4. |

### UI-A-02 — 빈칸(0) 개수가 2가 아님

| 항목 | 내용 |
|------|------|
| **테스트 이름 (예)** | `test_rejects_blank_count_not_two` |
| **Given** | 4×4이나 `0`이 1개 또는 3개(또는 0개)인 보드. |
| **When** | Boundary 입력 검증 호출. |
| **Then** | **`E_UI_BLANK_COUNT`**. |
| **보호 계약** | **BR-03 / AC-FR01-02**: `0`은 정확히 2칸. |

### UI-A-03 — 값 범위 위반 (0 및 1~16 외)

| 항목 | 내용 |
|------|------|
| **테스트 이름 (예)** | `test_rejects_value_out_of_allowed_range` |
| **Given** | 4×4, 빈칸 2개는 맞으나 셀에 `17`, `-1` 등 포함 (PRD **TD-05** 유형). |
| **When** | Boundary 입력 검증 호출. |
| **Then** | **`E_UI_VALUE_RANGE`**. |
| **보호 계약** | **BR-02 / AC-FR01-03**. |

### UI-A-04 — 0 제외 중복

| 항목 | 내용 |
|------|------|
| **테스트 이름 (예)** | `test_rejects_duplicate_nonzero_values` |
| **Given** | 4×4, 0은 2개, 그 외 동일 숫자 2회 이상 (PRD **TD-06** 유형). |
| **When** | Boundary 입력 검증 호출. |
| **Then** | **`E_UI_DUPLICATE`**. |
| **보호 계약** | **BR-04 / AC-FR01-04**. |

### UI-A-05 — 성공 경로에서 반환 배열 길이 6

| 항목 | 내용 |
|------|------|
| **테스트 이름 (예)** | `test_success_result_has_length_six` |
| **Given** | 입력 계약을 만족하는 4×4 (예: PRD **TD-01**). |
| **When** | 최종 `int[6]`을 반환하는 공개 API(오케스트레이터/경계) 호출. |
| **Then** | 성공 시 반환(또는 직렬화) 배열 **길이 == 6**. (구현 없으면 **항상 실패 = RED**.) |
| **보호 계약** | **BR-09 / AC-FR05-03 / PI-01**. |

### UI-A-06 — 반환 좌표 1-index

| 항목 | 내용 |
|------|------|
| **테스트 이름 (예)** | `test_success_coords_are_one_indexed` |
| **Given** | 유효 입력(예: **TD-01**), PRD 기대 1-index. |
| **When** | 동일 경로로 `int[6]` 획득. |
| **Then** | `r1,c1,r2,c2` **모두 1~4**. |
| **보호 계약** | **BR-10 / AC-FR05-04 / PI-02 / D-02**. |

**Track A RED:** Boundary·출력 계약이 없으면 위 테스트는 **의도적으로 전부 실패**한다.

## 5. TRACK B — Logic RED

전제: **유효한 4×4** 도메인 입력. I/O·CLI는 가정하지 않음. 대표 데이터는 PRD **TD-01~03**.

### LOG-B-01 — `find_blank_coords()`

| 항목 | 내용 |
|------|------|
| **테스트 이름 (예)** | `test_find_blank_coords_row_major_two_positions` |
| **Given** | **TD-01** `[[16,2,3,13],[0,0,10,8],[9,7,6,12],[4,14,15,1]]`. |
| **When** | `find_blank_coords(board)` |
| **Then** | row-major로 두 빈칸 **(2,1), (2,2)** in **1-index** (또는 팀이 고정한 0-index 쌍; PRD와 **한 가지**로 SSOT). |
| **Invariant** | **BR-05 / AC-FR02-01,02** |
| **RED** | 구현 없음 → 실패. |

### LOG-B-02 — `find_not_exist_nums()`

| 항목 | 내용 |
|------|------|
| **테스트 이름 (예)** | `test_find_not_exist_nums_returns_sorted_pair` |
| **Given** | **TD-01** |
| **When** | `find_not_exist_nums(board)` |
| **Then** | 누락 2개 **`(5, 11)`** (오름차순). |
| **Invariant** | **BR-06 / AC-FR03-01~03, PI-03** |
| **RED** | 구현 없음 → 실패. |

### LOG-B-03 — `is_magic_square()`

| 항목 | 내용 |
|------|------|
| **테스트 이름 (예)** | `test_is_magic_true_when_all_sums_34` / `test_is_magic_false_when_not_34` / (선택) `test_is_magic_rejects_if_zero_remains` |
| **Given** | (참) TD-01을 채운 0 없는 완성 보드. (거) 한 합만 깨진 0 없는 보드. (도메인) 0 잔존 보드. |
| **When** | `is_magic_square(board)` |
| **Then** | 10개 합 모두 34이면 `True`, 아니면 `False`; 0 잔존이면 **DomainValidationError** (PRD §7). |
| **Invariant** | **BR-07, FR-04** |
| **RED** | 구현 없음 → 실패. |

### LOG-B-04 — `solution()`

| 항목 | 내용 |
|------|------|
| **테스트 이름 (예)** | `test_solution_trial1_succeeds_td01` / `test_solution_trial2_succeeds_td02` / `test_solution_both_fail_td03` |
| **Given** | **TD-01** / **TD-02** / **TD-03** (PRD §11). |
| **When** | `solution(board)` |
| **Then** | TD-01: `[2,1,5,2,2,11]` (1-index). TD-02: `[1,1,16,4,4,1]`. TD-03: **SolveFailure** (Boundary는 `E_DOMAIN_NO_SOLUTION`, D-01). |
| **Invariant** | **FR-05, BR-08, BR-12, PI-01~05** |
| **RED** | 구현 없음 → 실패. |

**Track B RED:** 위는 모두 **의도적 실패**이며, UI RED와 **병행**한다.
